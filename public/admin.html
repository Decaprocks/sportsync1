📚 DBMS Concepts Implemented

EER Modeling

User → Player / Venue Manager specialization with total, disjoint constraints

BCNF Normalization

All tables decomposed to eliminate partial & transitive dependencies

Triggers

Eligibility check, ELO update, double booking prevention

Stored Procedures

Update_ELO, Book_Resource with transaction control

Transactions & ACID

START TRANSACTION, COMMIT, ROLLBACK, SELECT FOR UPDATE

Concurrency Control

Row-level locking prevents race conditions in bookings

Indexing (B-Tree)

Composite indexes on booking_date, player_location, skill_rating

Views

Leaderboard, recent matches, venue availability, player summary

Hybrid SQL–NoSQL

JSON document storage for unstructured reviews & feedback

Constraints

PK, FK (CASCADE/RESTRICT), UNIQUE, CHECK, NOT NULL enforced
